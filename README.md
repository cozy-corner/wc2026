# FIFA World Cup 2026 予想プロジェクト

北中米W杯(48カ国)の決勝トーナメントを、**地力(市場価値)＋疲労＋ホーム＋出場停止**で予想する。
このファイルは「データの出所と集め方」「更新手順」を残す手順書。

## ファイル構成 — 事実と予想を分離

| ファイル | 種別 | 内容 |
|---|---|---|
| `bracket.yaml` | 事実 | 会場マスタ(座標/標高/空調/7月気温湿度)、R32結果、R16〜決勝の組み合わせ・会場・日程・キックオフ、レッドカード |
| `group_stage.yaml` | 事実 | 全12グループ・72試合の結果・順位・突破フラグ |
| `strength.yaml` | 事実 | 全48チームの地力: 市場価値(Transfermarkt) + Elo(eloratings.net) |
| `suspensions.yaml` | 事実 | 公式カードに基づく出場停止(R16=R32レッド由来のみ) |
| `predictions.yaml` | 予想 | R16〜優勝の予想。`meta.version` で世代管理。事実は上記を参照 |
| `scripts/*.py` | ロジック | 疲労算出・相関・予想(R16→QF→SF→決勝) |

**原則**: 事実ファイルに予想を混ぜない。予想ファイルは事実を参照してよい。

## データソースと収集方法

すべて**2社以上でクロスチェック**し、確認できない項目は推測せず `UNCONFIRMED` とする。
食い違いは一次情報／整合計算で解決する(例: グループ勝点はスコアから再計算、
3位突破はR32の対戦カード実在で裏取り)。

### 試合結果・組み合わせ・会場・日程
- 主: Wikipedia「2026 FIFA World Cup knockout stage」/ 各グループページ(公式レポート出典)。
- 補: ESPN, CBS Sports, Yahoo Sports, olympics.com, FIFA.com。
- 会場は16開催都市。座標・標高・空調(開閉/固定屋根)・7月平年気温湿度は会場マスタに集約。

### キックオフ時刻(現地時間へ変換)
- Wikipedia各グループページは現地時刻を明記。ET表記しかない場合は会場のタイムゾーンで変換。
- ET→現地の時差(夏時間, メキシコはDST無しでUTC-6):
  - 太平洋(LA/Seattle/Santa Clara/Vancouver) = ET−3
  - 山岳 = ET−2 / 中部(Houston/Arlington/Kansas City) = ET−1
  - 東部(Boston/NJ/Atlanta/Toronto/Miami/Philadelphia) = ET−0
  - メキシコ(Mexico City/Guadalajara/Monterrey) = ET−2
- スクリプト側の変換表: `scripts/predict_qf.py` などの `ET_OFF`。
- 暑熱は「日中平均最高気温 × 時刻の日周補正 × 湿度」で試合ごとに算出(空調会場は0)。

### 地力(市場価値 + Elo)
- **市場価値**: Transfermarkt のスカッド総市場価値。planetfootball.com 経由(2026-07-01時点)、givemesport でクロスチェック。
- **Elo**: World Football Elo(eloratings.net)。**リーク/ネタバレ回避のため、ノックアウト前のスナップショット**
  (2026-07-01=グループ終了時、ライブ値はその後の試合を含むので不可)を Wayback Machine から取得。
- 両者を標準化してブレンド(相関重み: 市場価値0.48 / Elo0.52)。到達段階との相関は
  `scripts/strength_corr.py`(Spearman: 市場価値 rho≈0.66 / Elo rho≈0.71)。ブレンドは `scripts/blend.py`。

### カード・出場停止(公式のみ採用)
- 出典: FIFA Match Centre / Wikipedia各試合(公式レポート)。ESPN等で補強。
- **2026のルール**:
  - イエロー累積は**グループ終了後にリセット** → R32は全員クリーン。
  - 危険域は R32→R16→QF。異なる2試合でイエロー2枚 → 次戦停止。イエローはQF後に再リセット(決勝は累積停止なし)。
  - レッド(一発 or 1試合2枚目)は次戦停止、ストレート1試合分は上訴不可。
- **帰結**: R16の出場停止は「**R32でのレッド**」でしか発生しない(R16の前はR32の1試合だけ＝累積が成立しない)。
  → R32全16試合のレッドだけ洗えば十分。今大会は Balogun(USA)・Hincapié(Ecuador) の2枚のみ、
    進出チームは Balogun のみが該当(R16-6出場停止)。
- **怪我は非採用**: 公式で均一なデータが無く、非公式トラッカー依存はノイズが勝るため。
  カードのみ「公式・事実」として採用する。

### 収集の実務(大量データ)
- グループ72試合・R32会場・全キックオフ・市場価値・カードは**並行サブエージェント**で分担取得。
  - 例: グループを A-C / D-F / G-I / J-L に4分割、それぞれ結果・順位・キックオフを取得。
- サブエージェントには「WebSearch/WebFetchを使う」「ローカル検索はGrep/Glob/Read」「2社確認」
  「不明はUNCONFIRMED」を明示する(推測混入を防ぐ)。

## 予想モデル(要約)

実効クオリティ `Q = 地力 − λ·疲労 + ホーム − 出場停止減点`(地力=市場価値とEloのブレンド)、
勝率 `P = logistic((Qa−Qb)/T)`。パラメータは `predictions.yaml` の `meta.model`。
- 疲労 = 初戦からの累積(各試合の 出場時間 × 時刻補正暑熱、標高[順応国は割引]、総移動、休養、
  2連勝で首位確定した場合の第3戦ローテ)。詳細 `scripts/fatigue.py`。
- ホーム = 自国会場のみ(+0.45)、ホーム×高地>1500mで非順応の相手に追加(+0.5)。
- R16以降は延長なし前提。
- R32分析: 地力は結果をよく説明(rho 0.66)、疲労は二次的(拮抗カードのみ効く)。

## 更新手順(ラウンドが進んだら)

1. 実結果を**事実ファイル**に記録: `bracket.yaml`(該当ラウンドのスコア・レッド等)。
2. **カード確認**: そのラウンドのレッド、および(R16以降なら)イエロー累積で次戦停止者を
   公式出典で洗い、`suspensions.yaml` を更新。
3. `predictions.yaml` の該当予想を**確定値に置換**し、後段ラウンドを予想スクリプトで再計算。
4. `predictions.yaml` の `meta.version` を+1、`stage`・`changelog` を更新。

## スクリプト一覧
- `fatigue.py` — R16各チームの累積疲労。
- `strength_corr.py` — 市場価値 × 到達段階の相関。
- `r32_compare.py` — R32結果 vs R32前疲労の比較。
- `predict_r16.py` / `predict_qf.py` / `predict_sf.py` / `predict_final.py` — 各ラウンド予想。
- `inject_kickoffs.py` — 全試合にキックオフ現地時刻を注入(冪等)。
